## 1. 文档目的

本文档定义一期 MVP 上线后的首次迭代需求，聚焦员工自助能力与登录页体验优化。所有变更均建立在一期已完成的 8 个模块之上。

## 3. 功能详情

### 3.1 自助注册

#### 3.1.1 入口

- 登录页「登录」按钮下方新增「注册账号」链接或按钮。
- 点击后跳转至注册页或弹出注册对话框。

#### 3.1.2 注册表单字段

| 字段 | 必填 | 校验规则 |
|------|------|----------|
| 账号 | 是 | 1～64 字符，不可与已有账号重复 |
| 密码 | 是 | 6～64 字符 |
| 确认密码 | 是 | 须与密码一致 |
| 显示姓名 | 否 | 默认等于账号；最多 128 字符 |

#### 3.1.3 注册后行为

- 注册成功后自动登录并跳转至预约页（与登录后行为一致）。
- 注册失败：页面上提示具体错误（账号已存在、密码不一致等）。

#### 3.1.4 角色

- 新注册用户角色固定为 `EMPLOYEE`，不可自助注册为管理员。

### 3.2 去除测试账号提示

- 删除登录页底部 `<p class="login-hint">` 中的「测试账号：admin / admin123」文本。
- 不替换其他文本，不留空行。

### 3.3 员工修改显示姓名

#### 3.3.1 入口

- 点击顶部导航栏的用户名或「个人设置」进入修改。

#### 3.3.2 功能

- 当前显示姓名预填在输入框中。
- 修改后保存，顶部导航栏立即更新。
- 不允许为空，最长 128 字符。

### 3.4 员工修改密码

#### 3.4.1 入口

- 与修改显示姓名在同一页面或同一入口区域。

#### 3.4.2 表单字段

| 字段 | 必填 | 校验规则 |
|------|------|----------|
| 当前密码 | 是 | 须与数据库中当前用户密码匹配 |
| 新密码 | 是 | 6～64 字符 |
| 确认新密码 | 是 | 须与新密码一致 |

#### 3.4.3 修改后行为

- 密码修改成功后，提示用户重新登录（清除当前 Token，跳转登录页）。
- 失败时页面提示具体原因（当前密码错误、新密码不符合要求等）。

### 3.5 预约界面优化（时间与会议室选择合并）

#### 3.5.1 背景

一期预约流程分为三步：选时间 → 选会议室 → 确认。用户选完时间后需点击「下一步」才看到会议室列表，操作路径长。二期优化为选时间后即时展示可用会议室，减少一次页面跳转。

#### 3.5.2 新流程

原先的三步改为两步：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 第一步 | 选择日期与时段 | 上方为时间选择器，下方即刻展示该时段的空闲会议室列表 |
| 第一步（续） | 选择会议室 | 在同一页面点击选中一间会议室 |
| 第二步 | 填写主题并确认 | 确认预约信息，填写会议主题，提交 |

#### 3.5.3 交互行为

- 用户进入预约页，默认展示时间选择器和空闲会议室列表（空状态）。
- 选择或变更日期/时间后，自动重新查询并刷新下方会议室列表，无需手动点击「查询」或「下一步」。
- 点击会议室卡片选中后，该卡片高亮。
- 选中会议室后点击「下一步」进入确认页。
- 时间选择区和会议室列表在同一个`el-card`或区域中，左右或上下布局。

### 3.6 列表分页（通知 · 用户管理 · 会议室管理）

#### 3.6.1 背景

一期上述三个列表接口返回**全量数组**，数据量增大后页面加载慢、表格过长。二期与「我的预约」「预约管理」对齐，采用统一分页模型。

#### 3.6.2 适用范围

| 页面 | 路由 | 接口 |
|------|------|------|
| 通知中心 | `/notifications` | `GET /notifications` |
| 用户管理 | `/admin/users` | `GET /admin/users` |
| 会议室管理 | `/admin/rooms` | `GET /admin/rooms` |

#### 3.6.3 分页规则

| 项 | 规则 |
|----|------|
| 请求参数 | `page`，从 **1** 开始；缺省或 `<1` 时按第 1 页 |
| 每页条数 | 固定 **20**（与一期 `bookings/mine`、`admin/bookings` 一致） |
| 响应结构 | `{ items, page, pageSize, total }`（与一期分页一致） |
| 排序 | 通知：创建时间降序；用户：ID 升序；会议室：名称升序 |
| 前端组件 | `el-pagination`，`layout="prev, pager, next"`；仅当 `total > pageSize` 时显示 |

#### 3.6.4 交互行为

- 进入页面默认加载第 1 页。
- 切换页码后重新请求对应页数据。
- 删除/创建/编辑成功后刷新**当前页**；若当前页无数据且 `page > 1`，自动回退上一页（可选，与预约列表一致时实现）。
- 「全部标为已读」后刷新当前页并保持页码。

#### 3.6.5 兼容性说明

- **破坏性变更**：上述三个 `GET` 列表接口的 `data` 由数组改为分页对象；前端必须同步改造。
- 未读数量接口 `GET /notifications/unread-count` **不变**。

## 4. 安全约束

| 场景 | 行为 |
|------|------|
| 注册重复账号 | 返回明确错误提示，不暴露是否已注册 |
| 修改密码时当前密码错误 | 不允许修改，提示错误 |
| 注册时密码弱 | 前端校验长度 6-64，后端同样校验 |
| 分页越权 | 通知仅返回当前用户数据；用户/会议室列表仍仅 ADMIN |

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 一致性 | 分页参数、响应结构与一期预约列表一致 |
| 反馈 | 所有操作成功/失败均需提供明确的中文提示 |
| 性能 | 列表查询在数据库层分页（通知）；用户/会议室使用 MyBatis-Plus 分页 |
