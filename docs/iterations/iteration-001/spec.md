# 企业级会议室预约系统 — 二期迭代需求规格说明

| 文档版本 | 1.0 |
|----------|------|
| 状态     | 待确认 |
| 范围     | 二期小更新：自助注册 · 员工改密 · 员工改名 · 登录页优化 |

## 1. 文档目的

本文档定义一期 MVP 上线后的首次迭代需求，聚焦员工自助能力与登录页体验优化。所有变更均建立在一期已完成的 8 个模块之上。

---

## 2. 范围说明

### 2.1 本期包含

| 特性 | 说明 | 涉及模块 |
|------|------|----------|
| 自助注册 | 登录页增加注册入口，新用户可自助创建 EMPLOYEE 账号 | 认证、用户 |
| 去除测试账号提示 | 删除登录页底部「测试账号：admin / admin123」文本 | 登录页 UI |
| 员工修改显示姓名 | 员工可在系统内修改自己的显示名称 | 用户 |
| 员工修改密码 | 员工可在系统内修改自己的登录密码 | 认证、用户 |

### 2.2 明确不包含

- 邮件/短信验证码注册
- OAuth/第三方登录
- 手机号/邮箱绑定
- 个人头像上传
- 多语言支持

## 3. 功能详情

### 3.1 自助注册

#### 3.1.1 入口

- 登录页「登录」按钮下方新增「注册账号」链接或按钮。
- 点击后跳转至注册页或弹出注册对话框。

#### 3.1.2 注册表单字段

| 字段 | 必填 | 校验规则 |
|------|------|----------|
| 账号 | 是 | 1～64 字符，不可与已有账号重复 |
| 密码 | 是 | 6～64 字符 |
| 确认密码 | 是 | 须与密码一致 |
| 显示姓名 | 否 | 默认等于账号；最多 128 字符 |

#### 3.1.3 注册后行为

- 注册成功后自动登录并跳转至预约页（与登录后行为一致）。
- 注册失败：页面上提示具体错误（账号已存在、密码不一致等）。

#### 3.1.4 角色

- 新注册用户角色固定为 `EMPLOYEE`，不可自助注册为管理员。

### 3.2 去除测试账号提示

- 删除登录页底部 `<p class="login-hint">` 中的「测试账号：admin / admin123」文本。
- 不替换其他文本，不留空行。

### 3.3 员工修改显示姓名

#### 3.3.1 入口

- 点击顶部导航栏的用户名或「个人设置」进入修改。

#### 3.3.2 功能

- 当前显示姓名预填在输入框中。
- 修改后保存，顶部导航栏立即更新。
- 不允许为空，最长 128 字符。

### 3.4 员工修改密码

#### 3.4.1 入口

- 与修改显示姓名在同一页面或同一入口区域。

#### 3.4.2 表单字段

| 字段 | 必填 | 校验规则 |
|------|------|----------|
| 当前密码 | 是 | 须与数据库中当前用户密码匹配 |
| 新密码 | 是 | 6～64 字符 |
| 确认新密码 | 是 | 须与新密码一致 |

#### 3.4.3 修改后行为

- 密码修改成功后，提示用户重新登录（清除当前 Token，跳转登录页）。
- 失败时页面提示具体原因（当前密码错误、新密码不符合要求等）。

## 4. 安全约束

| 场景 | 行为 |
|------|------|
| 注册重复账号 | 返回明确错误提示，不暴露是否已注册 |
| 修改密码时当前密码错误 | 不允许修改，提示错误 |
| 注册时密码弱 | 前端校验长度 6-64，后端同样校验 |

---

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 一致性 | 注册、修改密码的校验规则与管理员创建用户一致 |
| 反馈 | 所有操作成功/失败均需提供明确的中文提示 |
