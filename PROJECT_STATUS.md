## 1. 项目名称

**企业级会议室预约系统**（Meeting Booking System）

---

## 2. 项目目标

帮助企业用统一、可视化的方式预约和管理会议室，减少微信群、Excel、口头预约带来的冲突和信息不透明。

**一期（MVP）要达成的效果：**

- 员工能在约 30 秒内完成一次标准预约（选时间 → 选房间 → 确认）。
- 系统能拦截时间冲突，用户无需线下协调。
- 管理员能独立完成会议室与用户的基础维护，无需开发介入。

## 7. 技术架构概览

用一句话说明：**浏览器访问 Vue 页面 → 调用 Spring Boot 接口 → 数据存在 MySQL 里。**

```
┌─────────────┐     HTTP/JSON      ┌──────────────────┐     SQL      ┌────────┐
│  Vue 3 前端  │  ───────────────►  │ Spring Boot 后端  │  ─────────►  │ MySQL  │
│ (浏览器)     │  JWT 认证          │ REST /api/v1      │  Flyway 迁移 │        │
└─────────────┘                    └──────────────────┘              └────────┘
```

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Vite + Element Plus | 单页应用，路由见下表 |
| 后端 | Spring Boot 2.7 + Java 8 | 单体应用，包：auth、user、room、booking、notification |
| 数据库 | MySQL 8 + Flyway | 表：用户、会议室、预约、通知 |
| 认证 | JWT + BCrypt | 登录后发 Token，后续请求携带 |
| 部署 | docker-compose（计划） | 本地一键启动前后端与数据库 |

## 9. 后端接口清单

| 方法 | 路径 | 说明 |

### 9.2 会议室

### 9.4 用户（管理员）

### 9.5 通知

## 10. 数据库 / 数据结构说明

| 表名 | 存什么 | 关键字段（通俗说明） |
|------|--------|----------------------|
| `sys_user` | 系统用户 | 账号、密码（加密）、显示名、角色（员工/管理员）、是否启用 |
| `meeting_room` | 会议室 | 名称、容量、楼层、类型、设备、状态（正常/维护中）、是否删除 |
| `booking` | 预约记录 | 哪间房、谁预约、会议主题、开始/结束时间、状态（有效/已取消） |
| `notification` | 站内通知 | 发给谁、类型、标题内容、关联预约、是否已读 |

**预约在页面上显示的状态**（不单独存库，按当前时间计算）：

- 未开始：可以取消
- 进行中 / 已结束：不能取消（一期规则）

**业务决策（plan ADR，已闭合 spec 待确认项）：**

- 会议室维护中：已有未来预约仍有效，仅禁止新预约。
- 删除会议室：若仍有未来预约，禁止删除。
- 我的预约：含历史已结束记录；未开始升序、已结束/已取消降序；每页 20 条。
- 创建用户最少字段：账号、密码、显示名、角色。
