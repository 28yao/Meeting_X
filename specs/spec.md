# 企业级会议室预约系统 — 产品需求规格说明

| 文档版本 | 1.0 |
|----------|-----|
| 状态     | 已确认（基于需求澄清问答） |
| 范围     | 单一办公地点 · 一期 MVP |

---

## 1. 文档目的

本文档定义「企业级会议室预约系统」一期（MVP）的产品需求，用于指导设计与开发。本文档仅描述**业务与产品行为**，不涉及技术实现方案。

---

## 2. 背景与目标

### 2.1 业务背景

企业内会议室资源需要通过统一、可视化的方式进行预约与管理，避免手工登记（微信群、Excel、口头预约）带来的冲突与信息不透明。

### 2.2 核心目标（一期优先）

1. **提高会议室利用率**：减少空置与浪费，使可用时段易于被发现和预约。
2. **提升占用可见性**：员工与管理员能清晰了解会议室在何时被占用、由谁占用。

### 2.3 成功标准（建议）

- 会议室预约冲突在系统内可被拦截，用户无需线下协调。
- 员工可在 30 秒内完成一次标准预约（选时间 → 选房间 → 确认）。
- 管理员可独立完成会议室与用户的基础维护，无需开发介入。

*具体量化指标（利用率提升百分比等）可在上线后根据运营数据补充。*

## 3. 范围说明

### 3.1 一期范围（MVP）

| 维度 | 说明 |
|------|------|
| 办公地点 | 单一办公地点/园区 |
| 预约生效 | 校验通过即生效，无需审批 |
| 预约路径 | 先选日期与时段，再选择有空闲的会议室 |
| 冲突规则 | 仅按起止时间重叠判定冲突 |
| 用户角色 | 普通员工、管理员 |
| 终端 | Web 端（浏览器访问） |

### 3.2 明确不在一期范围（二期或后续）

- 多办公地点/多城市
- 独立移动端 App
- 利用率及运营类报表
- 预约审批流
- 重复预约（周期例会）
- 与 Outlook / 企业日历等外部日历同步
- 操作审计日志
- 会议室按部门/职级限制使用
- 签到、爽约自动释放、候补队列

## 4. 用户与角色

### 4.1 角色定义

| 角色 | 描述 |
|------|------|
| **普通员工** | 为自己预约会议室；查看、取消自己的预约 |
| **管理员** | 维护会议室、用户账号、会议室状态；可管理全公司预约 |

### 4.2 账号与访问

- 用户**不能自助注册**。
- 由**管理员创建**用户账号并分配角色后，用户方可登录使用系统。
- 一期所有员工对全部会议室开放预约（无部门/职级限制）。

---

## 5. 会议室资源

### 5.1 会议室信息（一期展示与维护）

| 属性 | 说明 | 用户可见 |
|------|------|----------|
| 名称 | 会议室唯一标识名称 | 是 |
| 容纳人数 | 可容纳参会人数上限 | 是 |
| 楼层/位置 | 便于查找 | 是 |
| 类型 | 如：普通会议室、培训室、电话亭、高管室等 | 是 |
| 设备 | 如投影、白板、视频会议等，作为**会议室固定属性**展示 | 是（仅展示，预约时不单独勾选需求） |

*一期不支持在预约流程中按设备筛选会议室，也不支持在预约单中勾选「需要视频会议」等额外需求字段。*

### 5.2 会议室状态

| 状态 | 含义 | 是否可预约 |
|------|------|------------|
| 正常 | 可正常使用 | 是 |
| 维护中/临时关闭 | 由管理员设置 | 否 |

- 处于「维护中/临时关闭」的会议室，在预约流程中不可被选择，或选择后应明确提示不可预约。
- 普通员工**不能**修改会议室状态。

### 5.3 管理员对会议室的管理

- 新增、编辑、删除会议室信息。
- 设置会议室为「维护中/临时关闭」或恢复为「正常」。

---

## 6. 预约业务规则

### 6.1 预约主流程（普通员工）

```
选择日期 → 选择开始/结束时间（15 分钟粒度）→ 系统展示该时段空闲会议室列表 → 用户选择会议室 → 填写会议主题 → 确认提交 → 预约生效
```

**核心原则**：选择**目标会议时间**与**目标会议室**；若该会议室在所选时段无时间重叠的已有预约，且会议室为可预约状态，则预约成功。

### 6.2 时间规则

| 规则项 | 规定 |
|--------|------|
| 时间粒度 | 15 分钟（开始、结束时间均按 15 分钟对齐） |
| 可预约时段 | 全天 24 小时均可预约 |
| 跨天 | **不允许**；单次预约的开始与结束必须在**同一天**内 |
| 提前预约上限 | 最多提前 **30 天** |
| 重复预约 | 一期不支持 |

### 6.3 冲突判定

- **仅当**新预约与已有预约（含进行中、未开始）在时间上存在重叠时，视为冲突。
- **不**设置会前/会后缓冲时间（如前后 15 分钟预留）。
- 冲突时：**禁止提交**，向用户明确提示冲突，引导修改时间或更换会议室。
- 一期不提供推荐相近时段/其他房间、不提供候补队列。

### 6.4 并发预约（抢同一时段）

- 采用**先提交成功者得**的原则。
- 后提交者应收到明确的失败提示（该时段已被预约），需重新选择时间或会议室。

### 6.5 预约单信息

| 字段 | 必填 | 说明 |
|------|------|------|
| 会议主题 | 是 | 简短描述本次会议 |
| 会议室 | 是 | 流程中选择 |
| 开始时间 | 是 | 含日期 |
| 结束时间 | 是 | 含日期，须晚于开始时间 |
| 组织者 | 系统默认 | 默认为当前登录用户，一期不要求改选他人 |

*一期不要求填写：参会人数、参会人名单、备注、会议类型等。*

### 6.6 修改与取消

| 操作 | 普通员工 | 管理员 |
|------|----------|--------|
| 修改时间/会议室 | **不支持**（须取消后重新预约） | 可修改任意预约 |
| 取消预约 | 会议开始前**随时可取消**自己的预约 | 可取消任意预约 |

- 取消后，该时段释放，其他用户可预约。
- 一期不对爽约（预约后未实际使用）做自动处理；时段仍占用至预约结束时间止。

## 7. 可见性与隐私

### 7.1 日程信息可见性

当用户查看会议室占用情况（如在预约流程中查看某时段占用、或后续扩展的日程视图）时：

- 对**他人**的预约：展示 **会议主题**、**组织者**。
- 一期员工端不强制提供「全公司会议室日程看板」页面；若后续增加，仍遵循上述可见性规则。

### 7.2 自己的预约

- 在「我的预约」中可查看本人全部预约的完整信息（主题、时间、会议室、状态等）。

## 8. 功能清单（一期）

### 8.1 普通员工

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 登录 | 使用管理员分配的账号登录 | P0 |
| 预约会议室 | 按「选时间 → 选房间 → 填主题 → 确认」完成预约 | P0 |
| 我的预约 | 列表查看自己的预约；支持取消 | P0 |
| 站内通知 | 预约成功、预约被取消（含被管理员取消）时收到系统内通知 | P1 |

*一期不包含：会议室日程看板、日历多房间视图、会议室列表浏览页（可作为体验优化在二期评估）。*

### 8.2 管理员

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户管理 | 创建用户账号、分配角色（员工/管理员） | P0 |
| 会议室管理 | 增删改会议室信息及设备、类型等属性 | P0 |
| 会议室状态 | 设置维护中/临时关闭 | P0 |
| 预约管理 | 查看、修改、取消全公司任意预约 | P0 |
| 站内通知 | 同普通员工（与其操作相关的通知） | P1 |

## 9. 通知（一期）

| 事件 | 接收人 | 渠道 |
|------|--------|------|
| 预约成功 | 组织者 | 站内通知 |
| 用户自行取消 | 组织者 | 站内通知（可选，确认即记录） |
| 管理员取消/修改他人预约 | 原组织者 | 站内通知 |

- 一期**不**包含邮件、企业 IM（钉钉/企微/飞书）推送。

## 10. 典型用户故事

### US-01 员工预约会议

**作为** 普通员工，**我希望** 先选择会议日期和时段，再看到该时段所有可用的会议室，**以便** 快速完成预约并避免时间冲突。

**验收标准**：

- 只能预约未来 30 天内、15 分钟粒度对齐的时段。
- 不能预约跨天时段。
- 维护中的会议室不出现在可选列表或不可提交。
- 时间与已有预约重叠时无法提交，并看到明确错误提示。
- 提交成功后可在「我的预约」中看到记录，并收到站内通知。

### US-02 员工取消会议

**作为** 普通员工，**我希望** 在会议开始前随时取消自己的预约，**以便** 释放会议室供他人使用。

**验收标准**：

- 取消后该时段立即释放。
- 不能通过系统修改时间或换房，只能取消后重新预约。

### US-03 管理员维护会议室

**作为** 管理员，**我希望** 维护会议室信息与状态，**以便** 员工看到的资源准确可用。

**验收标准**：

- 可新增、编辑、删除会议室。
- 可将会议室设为维护中，此时员工无法成功预约该室。

### US-04 管理员处理异常预约

**作为** 管理员，**我希望** 修改或取消任意员工的预约，**以便** 处理突发情况或资源调配。

**验收标准**：

- 可修改预约的时间或会议室。
- 可取消任意预约，原组织者收到站内通知。

### US-05 管理员开通账号

**作为** 管理员，**我希望** 为员工创建登录账号，**以便** 其能使用预约系统。

**验收标准**：

- 可创建用户并指定为员工或管理员角色。
- 未创建账号的用户无法使用系统。

## 11. 边缘场景与产品行为

| 场景 | 期望行为 |
|------|----------|
| 预约结束时间早于或等于开始时间 | 禁止提交，提示时间无效 |
| 预约开始时间早于当前时间 | 禁止提交（不允许预约过去时段） |
| 预约超出 30 天范围 | 禁止提交，提示超出可预约范围 |
| 所选会议室在提交瞬间被他人抢先预约 | 提交失败，提示已被占用，请重选 |
| 管理员将会议室设为维护中，但该室已有未来预约 | 需定义：一期建议**已有预约仍有效**，新预约不可选择；或管理员先手动迁改/取消——*实现前与业务方确认默认策略* |
| 管理员删除仍有未来预约的会议室 | 需定义：禁止删除，或强制先处理预约——*建议禁止删除，提示先处理预约* |
| 用户取消已开始或已结束的预约 | 仅允许取消**未开始**的预约；已开始/已结束不可取消 |
| 会议进行中无人使用（爽约） | 一期不处理，占用至结束时间 |
| 两人预约相邻 15 分钟时段（无重叠） | 允许，视为无冲突 |
| 管理员修改他人预约 | 原组织者收到站内通知 |

## 12. 非功能需求（产品层面）

| 类别 | 要求 |
|------|------|
| 易用性 | 主流程步骤清晰，冲突与错误提示文案明确、可理解 |
| 一致性 | 时间展示、会议室状态、预约状态用语全站统一 |
| 可靠性 | 并发抢订场景下，不得出现双预约成功 |
| 可维护性 | 管理员可独立完成会议室与用户日常维护 |

---

## 13. 术语表

| 术语 | 定义 |
|------|------|
| 预约 | 用户为某会议室在指定时间段内的占用记录 |
| 冲突 | 新预约时段与已有预约时段存在时间重叠 |
| 组织者 | 创建该预约的用户，默认为预约人本人 |
| 维护中 | 管理员设置的会议室不可用状态 |
| 站内通知 | 系统内消息中心或通知列表，非邮件/IM |

---

## 14. 待确认项（实现前建议与业务方对齐）

1. **会议室进入维护时，已有未来预约的处理策略**（保留 / 批量取消 / 通知组织者）。
2. **删除会议室时，存在未来预约是否允许删除**及处理流程。
3. **「我的预约」列表** 默认排序（按开始时间升序/降序）及是否展示历史已结束预约。
4. **管理员创建用户** 时需采集的字段（姓名、工号、部门、邮箱等）最小集合。
